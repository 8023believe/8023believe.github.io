1. Android 10 安装

第三方比较完善的系统有lineageos和PixelExperienceos，通过在XDA网站可以找到手机对应的系统。（下载的系统为卡刷包）

  • reboot into recovery

With the device powered off, hold Volume Up + Power. Keep holding both buttons until the “MI” logo appears on the screen, then release.

进入recovery（第三方recovery）

  • Wipe

Now tap \Format Data** and continue with the formatting process. This will remove encryption and delete all files stored in the internal storage.

Return to the previous menu and tap \Advanced Wipe**, then select the Cache and System partitions and then \Swipe to Wipe**.

双清

  • Sideload the OS .zip package

    On the device, select “Advanced”, “ADB Sideload”, then swipe to begin sideload.

    On the host machine, sideload the package using: adb sideload filename.zip

在手机和电脑端进行操作进行卡刷（两端都要操作）

只操作电脑端可能报错

首先操作手机端，然后进行电脑端操作

卡刷完成

2. 证书获取

需要charles抓取手机https报文，则手机端必须安装charles证书（充当中间人）

  1. 下载证书

    手机WiFi设置http代理，通过浏览器访问chls.pro/ssl网址下载证书并安装。

    由于Android8以后，第三方证书只能安装在用户目录下，系统不信任，Charles不能代理https报文，所以需要将该证书移动到系统目录下。

  2. 证书改造

    系统证书目录为/system/etc/security/cacerts，证书命名格式为

​ 则必须改造原始证书，原始证书通过浏览器下载后一般在手机的/sdcard/Download目录下

​ 通过adb将该证书拉到本地处理，处理命令为openssl x509 -inform PEM -subject_hash_old -in xxxx.crt （Linux系统下一般都会有openssl命令）

得到新命名后将证书更名

mv charles-proxy-ssl-proxying-certificate.pem.crt *351def8d.0 *

3. 移动证书

将改造后的证书351def8d.0移动到系统证书目录下

通过adb push证书到/system/etc/security/cacerts目录下即可

但一般会遇到以下问题：

将证书移动到系统证书目录时会报系统只读

adb remount -> I get permission denied adb shell su -> I’m able to access shell and create folders etc and edit filesystem

Any help please

In case anyone has the same problem in the future:

Both adb remount and adb root don’t work on a production build without altering ro.secure, but you can still remount /system by opening a shell, asking for root permissions and typing the mount command.
